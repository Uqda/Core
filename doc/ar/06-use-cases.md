# حالات الاستخدام - شبكة عقدة

## 1. شبكة منزلية آمنة

### السيناريو

ربط جميع أجهزتك الشخصية بشكل آمن عبر الإنترنت.

```
🌐 الإنترنت
    ↓
📡 الراوتر (عليه عقدة)
    ↓
├── 💻 لابتوب (200:1111::1)
├── 📱 هاتف (200:2222::1)
├── 🖥️ كمبيوتر مكتبي (200:3333::1)
└── 📺 تلفاز ذكي (200:4444::1)
```

### الفوائد

- ✅ جميع الأجهزة تتواصل مباشرة
- ✅ مشفرة افتراضياً
- ✅ تعمل حتى خلف NAT
- ✅ الوصول لشبكة المنزل من أي مكان

### التكوين

```json
{
  "Listen": ["tls://[::]:9001"],
  "MulticastInterfaces": [
    {
      "Regex": ".*",
      "Beacon": true,
      "Listen": true,
      "Port": 9001
    }
  ]
}
```

## 2. شبكة موزعة عالمياً

### السيناريو

ربط الأصدقاء والعائلة حول العالم.

```
🇸🇾 سوريا ←→ 🇹🇷 تركيا ←→ 🇩🇪 ألمانيا
    ↓                          ↓
🇯🇴 الأردن                 🇳🇱 هولندا
```

### الفوائد

- ✅ اتصالات مباشرة، بدون وسطاء
- ✅ تجاوز الرقابة
- ✅ زمن انتقال منخفض (توجيه أقصر مسار)
- ✅ مقاومة لانقطاعات إقليمية

### التكوين

```json
{
  "Listen": ["tls://[::]:9001"],
  "Peers": [
    "tls://friend1.example.com:9001",
    "tls://friend2.example.com:9001",
    "tls://family.example.com:9001"
  ]
}
```

## 3. شبكة مجتمعية

### السيناريو

شبكة حي أو مجتمع محلي.

```
🏠 منزل 1 ←→ 🏢 شقة ←→ 🏠 منزل 2
    ↓            ↓              ↓
🏠 منزل 3 ←→ 🏪 متجر ←→ 🏫 مدرسة
```

### الفوائد

- ✅ مشاركة اتصال الإنترنت
- ✅ خدمات محلية (مشاركة الملفات، ألعاب)
- ✅ اتصالات طوارئ
- ✅ استقلالية المجتمع

### التكوين

```json
{
  "Listen": ["tls://[::]:9001"],
  "Peers": [
    "tls://neighbor1.local:9001",
    "tls://neighbor2.local:9001",
    "tls://community-gateway.local:9001"
  ],
  "MulticastInterfaces": [
    {
      "Regex": "wlan.*",
      "Beacon": true,
      "Listen": true,
      "Port": 9001
    }
  ]
}
```

## 4. شبكة طوارئ

### السيناريو

عند انقطاع الإنترنت أو في الكوارث.

```
📱 هاتف 1 ←→ 💻 لابتوب ←→ 📱 هاتف 2
  (WiFi)      (Ethernet)    (WiFi)
```

### الفوائد

- ✅ تعمل بدون إنترنت
- ✅ إعداد شبكة فوري
- ✅ لا تحتاج بنية تحتية
- ✅ حرجة في حالات الطوارئ

### التكوين

```json
{
  "Listen": ["tls://[::]:9001"],
  "MulticastInterfaces": [
    {
      "Regex": ".*",
      "Beacon": true,
      "Listen": true,
      "Port": 9001
    }
  ]
}
```

## 5. أجهزة IoT آمنة

### السيناريو

ربط أجهزة IoT بشكل آمن.

```
📹 كاميرا ←→ 🏠 محور ←→ 💡 أضواء
    ↓          ↓         ↓
🌡️ مستشعرات ←→ 🔐 قفل ←→ 🔊 مكبر صوت
```

### الفوائد

- ✅ لا اعتماد على السحابة
- ✅ تشفير من طرف لطرف
- ✅ معالجة محلية
- ✅ الخصوصية محفوظة

### التكوين

```json
{
  "Listen": ["tls://[::]:9001"],
  "Peers": [
    "tls://hub.local:9001"
  ],
  "AllowedPublicKeys": [
    "camera-key",
    "sensor-key",
    "lock-key"
  ]
}
```

## 6. VPN شخصي

### السيناريو

بديل لـ Tailscale/Zerotier.

```
💻 لابتوب ←→ 🌐 الإنترنت ←→ 🏠 خادم منزلي
```

### الفوائد

- ✅ بدون خوادم مركزية
- ✅ مشفر افتراضياً
- ✅ إصلاح ذاتي
- ✅ مجاني بالكامل

### التكوين

**على الخادم المنزلي:**

```json
{
  "Listen": ["tls://[::]:9001"],
  "Peers": []
}
```

**على اللابتوب:**

```json
{
  "Listen": ["tls://[::]:9001"],
  "Peers": [
    "tls://home-server.example.com:9001"
  ]
}
```

## 7. تجاوز الرقابة

### السيناريو

الوصول للمحتوى المقيد.

```
🇸🇾 سوريا ←→ 🇹🇷 تركيا ←→ 🌐 الإنترنت المفتوح
```

### الفوائد

- ✅ تجاوز القيود
- ✅ تصميم لامركزي
- ✅ صعب الحجب

### التكوين

```json
{
  "Listen": ["tls://[::]:9001"],
  "Peers": [
    "tls://free-peer1.example.com:9001",
    "tls://free-peer2.example.com:9001",
    "quic://free-peer3.example.com:9002"
  ]
}
```
