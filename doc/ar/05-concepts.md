# المفاهيم التقنية - شبكة عقدة

## كيف تعمل الشبكة؟

### 1. العناوين المشفرة (Cryptographic Addressing)

كل عقدة في شبكة عقدة تحصل على عنوان IPv6 مشتق من مفتاحها العام.

#### عملية توليد العنوان:

```
المفتاح الخاص (Private Key)
        ↓
المفتاح العام (Public Key) - Ed25519
        ↓
Hash (SHA-512)
        ↓
اقتطاع + إضافة بادئة
        ↓
عنوان IPv6 (200:xxxx:xxxx:xxxx::/64)
```

#### المزايا:

- ✅ **مستقل عن الموقع** - العنوان يتحرك مع الجهاز
- ✅ **لا يحتاج DHCP** - لا حاجة لخادم تخصيص عناوين
- ✅ **مستقر** - نفس العنوان طالما تستخدم نفس المفاتيح
- ✅ **آمن** - صعب جداً تزويره أو سرقته

### 2. نطاقات العناوين

#### عناوين العقد (Node Addresses): `200::/7`

```
مثال: 200:1234:5678:9abc:def0:1234:5678:9abc
```

- عنوان فريد لكل عقدة
- مشتق من المفتاح العام
- قابل للتوجيه على شبكة عقدة

#### الشبكات الفرعية (Subnets): `300::/7`

```
مثال: 300:1234:5678:9abc::/64
```

- شبكة فرعية واحدة (/64) لكل عقدة
- للإعلان للأجهزة الأخرى
- يمكن تخصيص عناوين لأجهزة غير عقدة

### 3. التوجيه الذكي (Smart Routing)

#### بناء الشجرة الممتدة (Spanning Tree)

```
         [Root Node]
        /     |     \
    [A]      [B]     [C]
   /  \      |      /  \
 [D]  [E]   [F]   [G]  [H]
```

**كيف تعمل:**

1. كل عقدة تعرف مسافتها من الـ root
2. المسارات تُحسب بناءً على هيكل الشجرة
3. الشجرة تُعاد بناءها تلقائياً عند تغيير البنية

#### اختيار المسار الأمثل

```
المصدر → الوجهة: مسارات متعددة متاحة

المسار 1: A → B → C → D (3 قفزات) ✅ المختار
المسار 2: A → E → F → G → D (4 قفزات)
المسار 3: A → H → I → J → K → D (5 قفزات)
```

**معايير الاختيار:**

- 🎯 أقصر مسار أولاً (أقل قفزات)
- ⚡ أقل زمن انتقال
- 📊 جودة الرابط
- 🔄 موازنة الحمل

#### التوجيه الجشع (Greedy Routing)

عندما لا يوجد مسار مباشر:

```
[المصدر] → [الأقرب للوجهة] → [التالي الأقرب] → [الوجهة]
```

- كل قفزة تقربك من الهدف
- العودة إلى توجيه الشجرة إذا لزم الأمر
- نهج هجين للأداء الأمثل

### 4. الإصلاح الذاتي (Self-Healing)

#### عند فشل رابط:

```
قبل الفشل:  A → B → C → D

           ↓ (B فشل)

بعد الفشل:  A → E → F → D
           (إعادة توجيه تلقائية)
```

**المزايا:**

- ⚡ وقت استرداد بالميلي ثانية
- 🔄 لا حاجة لتدخل يدوي
- 🎯 يجد دائماً أفضل مسار
- 💪 مقاوم للفشل

### 5. التشفير من طرف لطرف (End-to-End Encryption)

#### كيف يعمل:

```
[جهازك] 🔒 ← مشفر → 🔒 [جهازهم]
              ↓
      [عقد وسيطة]
    (لا تستطيع قراءة البيانات)
```

#### المكونات المشفرة:

**1. تبادل المفاتيح (Key Exchange):**

- Curve25519 ECDH
- تبادل آمن للمفاتيح

**2. اشتقاق مفاتيح الجلسة (Session Key Derivation):**

- HKDF
- مفاتيح فريدة لكل جلسة

**3. تشفير البيانات (Data Encryption):**

- ChaCha20-Poly1305
- تشفير عسكري المستوى

**4. تدوير المفاتيح (Key Rotation):**

- كل فترة زمنية
- Perfect Forward Secrecy (PFS)

#### ما الذي يحميه:

- ✅ **السرية** - محتوى البيانات مشفر
- ✅ **السلامة** - لا يمكن تعديل البيانات
- ✅ **المصادقة** - التحقق من المرسل
- ✅ **Forward Secrecy** - الجلسات السابقة آمنة

#### ما الذي لا يحميه:

- ❌ **إخفاء الهوية** - الأقران يمكنهم رؤية IP الخاص بك
- ❌ **البيانات الوصفية** - توقيت وحجم الحزم مرئي
- ❌ **تحليل حركة المرور** - أنماط الاتصال قابلة للملاحظة

## أنواع الاتصالات (Peering)

### 1. TCP

```
tcp://peer.example.com:9001
```

- اتصال TCP عادي
- ⚠️ غير مشفر
- ✅ بسيط وسريع

### 2. TLS

```
tls://peer.example.com:9001
```

- TCP مع تشفير TLS
- ✅ موصى به
- ✅ آمن ومشفر

### 3. QUIC

```
quic://peer.example.com:9002
```

- بروتوكول QUIC
- ✅ الأسرع
- ✅ مقاوم للفقدان

### 4. WebSocket

```
ws://peer.example.com:9001
wss://peer.example.com:9001
```

- WebSocket
- ✅ يعمل خلف بروكسي HTTP
- ✅ مفيد للشبكات المقيدة

### 5. SOCKS

```
socks://127.0.0.1:9050/hidden.onion:9001
```

- عبر بروكسي SOCKS
- ✅ للاتصال عبر Tor
- ✅ للاتصال عبر بروكسي آخر

### 6. UNIX Socket

```
unix:///var/run/uqda.sock
```

- اتصال محلي فقط
- ✅ سريع جداً
- ✅ آمن (محلي)

## الاكتشاف التلقائي (Multicast Discovery)

### كيف يعمل:

```
الجهاز A: "مرحباً! أنا أستخدم عقدة!"
الجهاز B: "أنا أيضاً! دعنا نتصل!"

         ↓

   [اتصال تلقائي]
```

### الخطوات:

1. 📢 البث على الشبكة المحلية
2. 👂 الاستماع لعقد عقدة أخرى
3. 🤝 إنشاء اتصالات تلقائية
4. ⚡ بدون أي تكوين

### متى يعمل:

- ✅ على نفس شبكة WiFi
- ✅ على نفس كابل Ethernet
- ✅ على نفس الشبكة المحلية (LAN)

### متى لا يعمل:

- ❌ عبر الإنترنت
- ❌ عبر VPN
- ❌ عبر NAT معقد

## الأمان والخصوصية

### ما الذي يحميه عقدة:

✅ **يحمي:**

- محتوى البيانات المُرسلة (مشفرة end-to-end)
- سلامة البيانات (لا يمكن تعديلها)
- هوية العقد (التحقق من المفاتيح العامة)

❌ **لا يحمي:**

- عنوان IP الحقيقي من الأقران المباشرين
- معلومات البيانات الوصفية (حجم الحزمة، التوقيت)
- الخدمات المفتوحة على العقدة (تحتاج firewall)

### توصيات الأمان:

#### 1. استخدام Firewall

```bash
# على Linux مع ip6tables
ip6tables -A INPUT -i uqda0 -m conntrack --ctstate ESTABLISHED,RELATED -j ACCEPT
ip6tables -A INPUT -i uqda0 -j DROP
```

#### 2. تحديد الأقران المسموح بهم

```json
{
  "AllowedPublicKeys": [
    "trusted-key-1",
    "trusted-key-2"
  ]
}
```

#### 3. استخدام TLS/QUIC

- ✅ `tls://` بدلاً من `tcp://`
- ✅ `quic://` للأداء الأفضل
- ✅ `wss://` بدلاً من `ws://`

#### 4. مراجعة الخدمات المفتوحة

```bash
# فحص الخدمات المفتوحة على عنوان IPv6 الخاص بك
netstat -tulpn | grep uqda0
```
