# الأمان والخصوصية - شبكة عقدة

## نظرة عامة على الأمان

### ما الذي يحميه عقدة

✅ **يحمي:**

- **محتوى البيانات** - مشفرة end-to-end
- **سلامة البيانات** - لا يمكن تعديلها
- **هوية العقد** - التحقق من المفاتيح العامة
- **Perfect Forward Secrecy** - الجلسات السابقة آمنة

❌ **لا يحمي:**

- **عنوان IP الحقيقي** - الأقران المباشرون يمكنهم رؤيته
- **البيانات الوصفية** - حجم الحزمة، التوقيت
- **تحليل حركة المرور** - أنماط الاتصال قابلة للملاحظة
- **الخدمات المفتوحة** - تحتاج firewall

## التشفير

### المكونات المشفرة المستخدمة

#### 1. Ed25519

- **الاستخدام:** التوقيعات الرقمية والمفاتيح العامة/الخاصة
- **المستوى:** عسكري
- **المزايا:** سريع وآمن

#### 2. Curve25519

- **الاستخدام:** تبادل المفاتيح (Key Exchange)
- **المستوى:** عسكري
- **المزايا:** ECDH آمن

#### 3. ChaCha20-Poly1305

- **الاستخدام:** تشفير البيانات
- **المستوى:** عسكري
- **المزايا:** سريع وآمن

#### 4. SHA-512

- **الاستخدام:** Hashing
- **المستوى:** آمن
- **المزايا:** مقاوم للتصادم

#### 5. HKDF

- **الاستخدام:** اشتقاق المفاتيح
- **المستوى:** آمن
- **المزايا:** اشتقاق آمن للمفاتيح

### عملية التشفير

```
1. تبادل المفاتيح (Curve25519 ECDH)
        ↓
2. اشتقاق مفاتيح الجلسة (HKDF)
        ↓
3. تشفير البيانات (ChaCha20-Poly1305)
        ↓
4. تدوير المفاتيح (كل فترة زمنية)
```

## أفضل الممارسات الأمنية

### 1. استخدام Firewall

#### Linux (ip6tables)

```bash
# السماح بالاتصالات القائمة
ip6tables -A INPUT -i uqda0 -m conntrack --ctstate ESTABLISHED,RELATED -j ACCEPT

# رفض كل شيء آخر
ip6tables -A INPUT -i uqda0 -j DROP

# حفظ القواعد
ip6tables-save > /etc/ip6tables.rules
```

#### Linux (ufw)

```bash
# حظر الوارد على واجهة عقدة
sudo ufw deny in on uqda0 proto ipv6
```

#### FreeBSD (ipfw)

```bash
# السماح بالاتصالات القائمة
ipfw add allow ipv6-icmp from any to me via uqda0

# رفض كل شيء آخر
ipfw add deny ipv6 from any to me via uqda0
```

### 2. تحديد الأقران المسموح بهم

```json
{
  "AllowedPublicKeys": [
    "trusted-peer-1-public-key",
    "trusted-peer-2-public-key"
  ]
}
```

**الاستخدام:** لإنشاء شبكة خاصة - فقط هذه المفاتيح يمكنها الاتصال بك

### 3. استخدام TLS/QUIC

**✅ جيد:**

```json
{
  "Listen": ["tls://[::]:9001"],
  "Peers": ["tls://peer.example.com:9001"]
}
```

**❌ سيء:**

```json
{
  "Listen": ["tcp://[::]:9001"],
  "Peers": ["tcp://peer.example.com:9001"]
}
```

### 4. حماية المفتاح الخاص

```bash
# صلاحيات الملف
chmod 600 uqda.conf

# ملكية الملف
chown root:root uqda.conf

# أو تخزين في ملف منفصل
uqda -useconffile uqda.conf -exportkey > /etc/uqda/private.key
chmod 600 /etc/uqda/private.key
```

### 5. استخدام كلمات مرور للأقران

```json
{
  "Peers": [
    "tls://peer.example.com:9001?password=strongPassword123"
  ]
}
```

**⚠️ يجب أن يكون نفس كلمة المرور على كلا الجانبين**

### 6. تثبيت المفاتيح العامة

```json
{
  "Peers": [
    "tls://peer.example.com:9001?key=expected-public-key"
  ]
}
```

**الاستخدام:** للتحقق من هوية الـ peer - الاتصال يفشل إذا كان المفتاح لا يطابق

### 7. مراقبة العقدة

```bash
# فحص الاتصالات النشطة
uqdactl getPeers

# فحص الجلسات
uqdactl getSessions

# مراقبة السجلات
journalctl -u uqda -f
```

## الخصوصية

### ⚠️ عقدة ليست شبكة مجهولة

**ما يعنيه هذا:**

- الأقران المباشرون يمكنهم رؤية عنوان IP الحقيقي
- ليس مثل Tor أو I2P
- الهدف هو **الأمان** وليس **إخفاء الهوية**

### إذا كنت تحتاج إخفاء الهوية

**استخدم Tor:**

```json
{
  "Peers": [
    "socks://127.0.0.1:9050/hidden.onion:9001"
  ]
}
```

**إعداد Tor:**

```
# /etc/tor/torrc
IsolateSOCKSAuth 1
```

### حماية البيانات الوصفية

**البيانات الوصفية المرئية:**

- حجم الحزم
- توقيت الحزم
- أنماط الاتصال
- عدد الاتصالات

**للحماية:**

- استخدم Tor للاتصالات
- استخدم VPN تقليدي
- تجنب الاتصال بأقران غير موثوقين

## التهديدات المحتملة

### 1. هجمات Man-in-the-Middle

**الحماية:**

- ✅ استخدام TLS/QUIC
- ✅ تثبيت المفاتيح العامة
- ✅ التحقق من المفاتيح يدوياً

### 2. هجمات DDoS

**الحماية:**

- ✅ استخدام Firewall
- ✅ تحديد AllowedPublicKeys
- ✅ Rate limiting (في المستقبل)

### 3. تسريب المفتاح الخاص

**الحماية:**

- ✅ حماية الملف بصلاحيات مناسبة
- ✅ عدم مشاركة المفتاح
- ✅ نسخ احتياطية مشفرة

### 4. تحليل حركة المرور

**الحماية:**

- ✅ استخدام Tor
- ✅ تشفير إضافي على طبقة التطبيق
- ✅ تجنب الاتصال بأقران غير موثوقين

## التوصيات الأمنية حسب السيناريو

```json
{
  "AllowedPublicKeys": [
    "device-1-key",
    "device-2-key",
    "device-3-key"
  ],
  "NodeInfoPrivacy": true
```

```json

}
```

**⚠️ كن حذراً:** الاتصال بأقران عامين يجعل شبكتك مرئية

```json
      "Port": 9001,
      "Password": "emergency-password"
    }
  ]
}
```
